# Page Brief — Q&A (Cited Answers)

> Inherits `../design-system.md`. Route: `/modules/qa`. Module 8 (AGENT-09), Wave 4.
> Covers FR-039..041.

## Purpose

A focused question-answering surface over the whole knowledge base, returning a concise,
**cited** answer (distinct from the conversational dock — this is a "search → answer with
sources" page, closer to an enterprise answer engine).

## Content & data

- **Ask box** (prominent, hero).
- **Answer:** a synthesized, streamed answer with inline/numbered **citations** mapping to
  vault files (`file_path`, content_type, score).
- **Sources panel:** the cited assets as cards, in rank order.
- **Follow-ups:** suggested next questions.

## Primary actions

Ask · read the cited answer · open a source · ask a follow-up · copy the answer.

## States

Empty (just the ask box + example questions) · answering (streaming) · answered (answer +
sources) · no-confident-answer · error.

## Layout

Hero ask box centered (collapses to top after asking). Answer in the main column with
citation markers; a right "Sources" rail listing cited assets. Follow-up chips beneath.

## Design prompt seed

> Design a "Q&A" answer-engine page for an enterprise AI-consulting platform. A prominent
> centered ask box (with a few example questions below it) that, after asking, moves to
> the top. The answered state shows a synthesized, well-formatted answer in the main
> column with numbered inline citation markers, and a right-hand "Sources" rail listing
> the cited vault documents as ranked cards (title, content-type badge, relevance). Add
> suggested follow-up question chips beneath the answer and a "copy answer" action. Show
> the empty state, a streaming/answering state, and the answered state. Clean, trustworthy
> answer-engine aesthetic (think a calm enterprise Perplexity); indigo accent; Inter;
> rounded-lg; light + dark. Use a generic "How do we handle PII in a RAG pipeline?"
> example — no company names.
